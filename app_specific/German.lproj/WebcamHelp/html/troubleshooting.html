Fehlersuche

Allgemeine Hinweise

Bitte befolgen Sie diese Hinweise wenn Sie auf ein Problem treffen, das entweder nicht separat erwähnt wird oder wenn die Hinweise dort das Problem nicht lösen:

Überprüfen Sie die USB-Kabelverbindungen. Starten Sie macam neu. Zeihen Sie die Kamera ab und schließen Sie sie neu an. Stellen Sie sicher, daß Sie die neuste Version von Mac OS X benutzen. Schließen Sie die Kamera direkt (nicht über einen Hub) an einen eingebauten USB-Anschluß Ihres Rechners an (nicht an eine USB-Hostkarte eines anderen Herstellers). Lesen Sie die separate Dokumentation und eventuelle Hinweise am macam Netzort, ob es sich um ein bekanntes Problem handelt. Lesen Sie weiter unten, was sie in einem Fehlerreport angeben sollten.

Problem: Kein Video, keine Mitteilung

Überprüfen Sie, ob es sich um eine unterstützte Kamera handelt. Eine Liste unterstützter Kameras befindet sich am macam Netzort.

Mitteilung: Kamera wird benutzt

Beenden Sie andere Programme. Beenden Sie die "Classic"-Umgebung. Ziehen Sie die Kamera ab und schließen Sie sie wieder an. Starten Sie macam neu.

Mitteilung: Nicht genügend Strom

Schließen Sie Ihre Kamera direkt an Ihrem Rechner oder an einem USB-Hub, der über eine eigene Stromversorgung verfügt, an.

Mitteilung: Nicht genügend Bandbreite

Ziehen Sie andere USB-Geräte ab oder verwenden Sie für diese Geräte einen anderen USB-Anschluß Ihres Rechners. Wenn bei Ihrer Kamera möglich, verkleinern Sie das Bildformat, reduzieren Sie die Bildrate und/oder erhöhen Sie die Datenkompression.

Mitteilung: Prozessor ausgelastet

Versuchen Sie es noch einmal. Wenn Sie auf dieses Problem regelmäßig stoßen: Beenden Sie andere Programme, tun Sie weniger Dinge, die Rechenkapazität brauchen. Benutzen Sie weniger Kameras gleichzeitig. Verringern Sie die Bildgröße oder Bildrate. Kaufen Sie einen schnelleren Macintosh...

Mitteilung: Nicht genügend RAM

Starten Sie macam oder Ihr Videoprogramm neu. Beenden Sie andere Anwendungen. Stecken Sie mehr RAM in Ihren Rechner.

Mitteilung: Nicht unterstützt

Ihre Kamera wird zur Zeit nicht unterstützt. Sehen Sie in der Liste unterstützter Kameras am macam Netzort nach genaueren Informationen nach.

Tonprobleme

Audio wird nicht von macam sondern von den eingebauten USB Audio-Treibern von Mac OS X verarbeitet. Lesen Sie die Dokumentation dort.

Installionsprobleme bei macam.component

Bitte stellen Sie sicher, daß Sie den Ordner "QuickTime" im Ordner "Library" direkt auf Ihrer Festplatte benutzen (nicht verwechseln mit "Library"-Ordnern an anderen Orten). Stellen Sie sicher, daß Sie Administrationsrechte an dem betroffenen Rechner haben. Bewegen Sie ältere Versionen von macam.component in den Papierkorb, bevor Sie die neue Version installieren (Wenn Sie Probleme beim Löschen der alten Komponente haben: Nach einem Neustart funktioniert es). Versuchen Sie, macam ohne andere Komponenten laufen zu lassen (bewegen Sie sie vorübergehend an einen anderen Ort und starten Sie den Rechner neu).

Prgrammabstürze, "Kernel Panics", andere eigenartige Dinge

Kontaktieren Sie den Autor. Geben Sie detaillierte Informationen. Helfen Sie bei der Fehlerbehebug. Warten Sie auf die nächste Version...

Fehler mit der QuickTime-Komponente

Versuchen Sie, das Problem mit einem anderen Programm zu wiederholen. Wenn sich das Problem dort wiederholt, liegt es an der Komponente. Wenn das Problem nur mit einem bestimmten Programm auftritt, könnte es auch ein Fehler des betroffenen Programms sein. Bitte erkundigen Sie sich in der Dokumentation desjenigen Programms, ob es sich um einen bekannten Fehler handelt. Es könnte dennoch ein Fehler von macam sein. Kontaktieren Sie deshalb bitte beide Autoren.

Fehlerberichte Verschicken

Wenn Sie auf etwas gestoßen sind, das nach einem Fehler in macam aussieht, ist Ihre Mithilfe gefragt: Bitte teilen Sie den Fehler dem Autoren mit. Einige Informationen sind notwendig, um den Fehler effektiv finden und beheben zu können. Versuchen Sie, den Fehler zu isolieren. Wenn es sich um einen Programmabsturz handelt, schalten Sie "Absturzinformationen sichern" in den Voreinstellungen von Console.app ein und wiederholen Sie die Absturzsituation. Sie erhalten einen Absturzbericht, den Sie bitte an Ihren Fehlerreport anhängen - ebenso wie Meldungen in der Konsole, die seit Programmstart dort erschienen sind. Andere Informationen, die Sie hinzufügen sollten, sind:

  • Ihr Macintosh-Modell, Prozessor(en), Takt, RAM, Erweiterungen
  • Die Mac OS X-Version, mit der Sie arbeiten
  • Besondere Erweiterungen und Zusatzgeräte
  • Andere USB-Geräte
